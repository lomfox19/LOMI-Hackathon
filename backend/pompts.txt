PROJECT RULES FOR CODE GENERATION

Before generating any code, read the design guidelines located at:

src/design-system/ui-guidelines.md

Follow all design rules defined in that file.

STRICT REQUIREMENTS

1. Do NOT modify existing code, files, or logic in the project.
2. Do NOT change previously implemented components, APIs, or database schemas.
3. Only create NEW components, pages, APIs, or services required for the requested feature.
4. All frontend components must follow the UI theme, color palette, typography, and animation rules from the design guidelines.
5. Use existing reusable components when possible.

FRONTEND RULES

* Follow the AI SaaS theme defined in the design guidelines.
* Use React + Tailwind CSS.
* Use Framer Motion for animations.
* Maintain consistent layout, spacing, and color system.

BACKEND RULES

* Backend APIs should match the requirements of the new frontend feature.
* Backend code must integrate with existing APIs without modifying previous routes or logic.
* Follow the existing backend structure when creating new controllers, routes, or services.

SAFETY RULES

If implementing the requested feature requires modifying existing code:

1. STOP immediately.
2. Explain what changes are required.
3. Ask for confirmation before modifying anything.

Never break existing functionality.

If the task cannot be completed without affecting the current system, return the issue instead of modifying the existing environment.

GOAL

Generate only the necessary new files and code required to implement the requested feature while keeping the existing system stable and unchanged.
